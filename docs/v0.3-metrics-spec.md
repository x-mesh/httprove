# httprove v0.3 메트릭 노출 구현 설계 (트랙1 8종 통합)

검증(verify)의 high/medium 이슈를 모두 반영한 최종본. 대상 8종: **B5**(TLS info), **B6**(cert chain), **B7**(throughput), **B9**(reuse/version), **B11**(DNS), **B12**(fleet rollup), **B13**(verdict), **B14**(Server-Timing).

핵심 교정 요약:
- **B13 stateset → gauge**: exporter가 `text/plain; version=0.0.4`(Prometheus 포맷, OpenMetrics 아님)로 서빙하므로 `stateset`/`info` TYPE는 무효. B13·B5 모두 enum-gauge(현재=1.0, 나머지=0.0 / 값 항상 1.0)로 출력.
- **B12 changed_files에 `src/lib.rs` 누락 보강**: `--prom` 경로의 `TargetState`도 성공 시에만 `last_success`를 저장 → exporter와 동일 한계. `down: bool`을 두 호출처(lib.rs `consume`, exporter update task) 모두에서 precompute해야 컴파일·정합.
- **B12 no-target-label 예외 명시 비준**: `httprove_targets_*`, `httprove_fleet_*`는 'target 레이블 필수' 규칙의 의도적 예외(fleet 스칼라/agg 축). 명시 승인 필요.
- **B11 output_source `render_rollup` → `render`** 오타 교정. 재사용 hop은 `resolved_ips`를 승계(빈 집합 아님) → 가드는 방어용.

## 1. 공통 인프라 변경

### 1-A. TargetMetrics 필드 확장 (B12, B13 — 한 번에 처리)

`render-only`로는 산출 불가한 두 메트릭만 필드를 추가한다. **두 필드를 한 PR에서 같이 추가**해 호출부를 한 번만 손댄다.

```rust
// src/output/prom.rs
pub struct TargetMetrics<'a> {
    pub target: &'a str,
    pub stats: &'a StatsCollector,
    pub last_success: Option<&'a ProbeResult>,
    /// B13: 호출처에서 verdict::assess(latest, &vctx).state로 precompute. None => 섹션 생략.
    pub verdict_state: Option<VerdictState>,
    /// B12: latest 관측이 DOWN인지. 호출처 precompute(warmup/no-result => false).
    pub down: bool,
}
```

**왜 호출처 precompute인가**: `last_success`는 성공 시에만 set되므로 `None`이 'never probed'인지 'currently failing'인지 render에서 구분 불가. verdict/down 판정은 error를 포함한 *최신* ProbeResult가 필요한데 이는 호출처(lib.rs `consume`, exporter update task)만 보유.

**공용 헬퍼 추가**(로직 드리프트 방지, lib.rs/exporter.rs 양쪽 동일):
```rust
// src/verdict.rs (또는 types.rs) — verdict.rs:81/:102 의미와 일치
pub fn is_down(r: &ProbeResult) -> bool {
    !r.is_success() || r.final_hop().is_none()
}
```

**호출부 변경 3곳**:

1. **lib.rs `TargetState` 확장** (구조체에 `latest_down: bool`, `latest_state: Option<VerdictState>` 추가, init=false/None):
   - `consume` 클로저(lib.rs `targets[i].stats.record(&result)` 직후, `if result.is_success()` 갱신 *전*)에서 **모든 결과마다**:
     ```rust
     targets[i].latest_down = verdict::is_down(&result);
     targets[i].latest_state = Some(verdict::assess(&result, &vctx).state);
     ```
   - `--prom` 매핑(lib.rs:442~)에 `down: t.latest_down, verdict_state: t.latest_state` 추가.
   - `vctx`는 lib.rs:342에서 이미 빌드됨(warn/cert_warn_days, baseline=None).

2. **exporter.rs SharedState 5-tuple로 확장**(또는 named struct 권장):
   ```rust
   type SharedState = Arc<Mutex<Vec<(String, StatsCollector, Option<ProbeResult>, bool, Option<VerdictState>)>>>;
   ```
   - update task(exporter.rs:88~)에서 매 결과마다 `slot.3 = is_down(&result); slot.4 = Some(assess(&result,&vctx).state);`. vctx는 `run_exporter`의 `out_cfg`(Copy)로 빌드.
   - `handle_connection` 시그니처에 `vctx`/`out_cfg` 불필요(상태 슬롯에 precompute됨). Route::Metrics 매핑(exporter.rs:155~)에 `down`/`verdict_state` 추가.

3. **기존 prom.rs 테스트의 모든 `TargetMetrics{...}` 리터럴**(6곳: prom.rs:318/351/400/405/444/468 근방)에 `verdict_state: None, down: false` 추가 — 안 하면 컴파일 실패. 구현 체크리스트 항목.

### 1-B. StatsCollector 누적 확장 (B7, B9, B11)

세 메트릭이 누적 분포/카운터를 요구. 모두 `record()`의 **성공 분기**(실패 early-return 이후)에 누적, getter 추가, `reset()`은 `*self=Self::new()`라 신규 Default 필드 자동 클리어. `summed_timings()`는 record 내 기존 `let t = result.summed_timings()`(stats.rs:185) 재사용.

| 메트릭 | 신규 필드 | record 누적 | getter |
|---|---|---|---|
| B7 throughput | `throughput: PhaseAccum` | body≥4096 && download_ms>0 일 때 `throughput.push(body / (download_ms/1000.0))` | `throughput_stats() -> Option<PhaseStats>` |
| B9 reuse/version | `hops_total: u64`, `hops_reused: u64`, `http_version_counts: BTreeMap<String,u64>` | hop 1회 순회: `hops_total += hops.len()`; reused 카운트; `normalize_http_version` 슬러그 누적 | `hops_total()`, `hops_reused()`, `http_version_counts()` |
| B11 DNS | `last_sorted_ips: Option<Vec<String>>`, `dns_answer_changes: u64` | `final_hop().resolved_ips` 정렬→직전과 비교, 다르면 +1, baseline 갱신(`!cur.is_empty()` 가드) | `dns_answer_changes()` |

`normalize_http_version(v)`: `"HTTP/2"→"http2"`, `"HTTP/1.1"→"http1.1"`, `"HTTP/1.0"→"http1.0"`, `"HTTP/3"→"http3"`, `"HTTP/0.9"→"http0.9"`, `"HTTP/?"→"unknown"`(아래 §6 결정). `http_version_str`(probe.rs:1056)이 항상 `HTTP/` prefix를 반환하므로 6종 전부 커버.

### 1-C. 새 공용 모듈/함수 — 추가 불필요

- **Server-Timing 파서(B14)**: `otlp::parse_server_timing(&[(String,String)]) -> Vec<(String,Option<f64>)>`(otlp.rs:69)가 이미 `pub`·순수. prom.rs에서 `use crate::otlp::parse_server_timing;`로 그대로 호출. (otlp는 `output` 모듈의 형제, 절대경로 import 정상. lib.rs:389에 동일 호출 선례.)
- **ChainAnalysis 동기 생성기(B6)**: `chain::analyze(&[CertInfo]) -> ChainAnalysis`(chain.rs:89)가 순수·동기·패닉 없음(빈 체인=default). render에서 직접 호출. 네트워크 경로(`check_aia`, async)는 사용 안 함. analyze 호출 선례는 text.rs:457.

신규 공용 코드는 §1-A의 `is_down` 헬퍼 하나뿐.

- B14 inf/NaN/음수 → `v.is_finite() && v >= 0.0` 가드 필수(rustc로 inf/-5 파싱 확인).

**사용자 결정 필요**:

1. **(B12) no-target-label 예외 비준** — `httprove_targets_total/down`, `httprove_fleet_phase_milliseconds`는 '모든 메트릭 target 레이블 필수' 규칙의 의도적 예외. 명시 승인 요청. (대안: fleet에 `target="__all__"` 합성행 → 실제 타깃 URL과 충돌 위험으로 비권장.)

2. **(B14) 분포 vs 스냅샷** — 권장=스냅샷 게이지(last_success). 분포(p50/p95) 원하면 StatsCollector에 `BTreeMap<String,PhaseAccum>` 추가가 필요한데, **Server-Timing metric 이름이 서버 제어·무제한**이라 카디널리티 폭발 위험 → flag + allow-list 게이팅 전제. 현재는 스냅샷 채택. 분포가 필요한지 확인.

3. **(B12) rollup 레이블 방식** — `agg=worst|best` 채택(per-target `stat=min|max`와 축 혼동 회피). `agg=max|min` 선호 시 변경. fleet 백분위는 per-target 백분위의 max/min 근사(병합 분포 아님) — HELP에 명시.

4. **(B9) http_version 정규화** — `"HTTP/?"`/HTTP/0.9 → `"unknown"` vs `"http?"`. 권장=`"unknown"`(라벨 가독성). 슬러그는 dot 보존(`http1.1`/`http2`, http_version_str와 1:1) — `http_1_1` 미채택.

5. **(B11) resolved_ip 출처 hop** — 현재 gauge·counter 모두 `final_hop`(리다이렉트 도착 호스트의 DNS). '원본 target의 A/AAAA'를 원하면 `hops.first()`로 변경. 멀티 CDN 리다이렉트는 final 기준이면 answer_changed가 의도 외로 증가 가능. HELP에 `(final hop)` 명시로 일단 진행, 검토 요청.

6. **(B7) goodput 임계·단위** — `MIN_GOODPUT_BODY_BYTES=4096`(한 페이지), `bytes_per_second`(SI). 8192/16384 또는 Mbit/s 선호 시 변경. download는 체인 합산 — HELP에 `(sum over hops)` 명기.

## 7. 네이밍 일관성 점검

**prefix/suffix 규칙** — 전부 준수:
- `httprove_` prefix: 18개 신규 메트릭 전부 ✓
- counter `_total`(단조): `hops_total`, `connection_reused_total`, `http_version_total`, `dns_answer_changed_total`. 전부 **StatsCollector 누적** 출처 → 단조 보장 ✓. last_success 기반은 counter 없음 ✓.
- 시간 `_milliseconds`: `server_timing_milliseconds`, `fleet_phase_milliseconds` ✓. (`_days`는 도메인 단위로 예외 허용 — 기존 `cert_expiry_days`와 일관, `weakest_expiry_days` ✓).
- ratio 0~1: `connection_reuse_ratio` ✓ (분모 0 가드 `total==0 → 생략`).
- bytes: `throughput_bytes_per_second`(SI base) ✓.
- enum-gauge(값 1.0 + 식별 레이블): `tls_info`(값1.0), `verdict_state`(현재=1/나머지=0), `target_up`(0/1) ✓.

**충돌/중복 검사**:
- 18개 신규 이름 전부 고유, 기존 메트릭(`probes_total`/`probe_failures_total`/`expect_failures_total`/`status_total`/`phase_milliseconds`/`last_total_milliseconds`/`last_body_bytes`/`cert_expiry_days`)과 무충돌 ✓.
- B6 `cert_chain_depth/incomplete/weakest_expiry_days`는 기존 `cert_expiry_days`(leaf only)와 별개 시리즈 — 의미 구분 명확(leaf vs 전체 체인) ✓.
- `target="__all__"` 합성행 미사용 → fleet 메트릭은 별도 이름+무target레이블이라 실제 타깃 URL과 충돌 불가 ✓.

**레이블 일관성**:
- `target` 레이블: fleet/scalar 3종 제외 전부 보유. 예외는 §6.1 비준 대상.
- `stat` 레이블 값: STAT_ORDER `[min,mean,p50,p95,p99,max]` 고정(B7/B12 phase 메트릭과 기존 `phase_milliseconds` 동일).
- `state`(B13 verdict): 소문자 `pass|degraded|down`(VerdictState 선언 순서). `version`(B9): `http1.1|http2|...`. `agg`(B12): `worst|best`. `phase`(B12): `phase.label()` = `dns|tcp|tls|ttfb|download|total`.
